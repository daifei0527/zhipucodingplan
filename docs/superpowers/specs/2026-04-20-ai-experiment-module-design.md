# AI实验模块设计文档

## 概述

在抢购过程中集成远程大模型服务，实时分析异常情况并执行探索性实验，实验结果用于改进下次抢购代码，不影响主进程抢购流程。

## 架构

```
┌─────────────────────────────────────────────────────────────┐
│                      buyer/purchase.py                       │
│  ┌─────────────────────────────────────────────────────┐    │
│  │              抢购主循环 (100ms轮询)                    │    │
│  │                                                     │    │
│  │   请求 → 响应 → 检查事件 → [关键事件?] → 触发实验模块   │    │
│  │                      ↓                              │    │
│  │                  正常流程继续                         │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                            ↓ 异步触发
┌─────────────────────────────────────────────────────────────┐
│                   ai_lab/experiment.py                       │
│                                                             │
│  1. 收集上下文 (最近请求/响应/库存状态)                        │
│  2. 调用大模型分析                                           │
│  3. 生成实验方案 (1-3个尝试)                                  │
│  4. 执行实验请求                                             │
│  5. 记录结果到 logs/ai_experiments.json                      │
│  6. 生成改进建议                                             │
└─────────────────────────────────────────────────────────────┘
```

## 关键事件触发

| 事件类型 | 触发条件 | 实验方向 |
|---------|---------|---------|
| `rate_limit` | 响应提示限流 | 尝试不同请求间隔、不同API端点 |
| `captcha` | 检测到验证码 | 探索绕过方法、分析触发原因 |
| `auth_error` | 认证失败 | 尝试刷新token、探索认证机制 |
| `inventory_found` | 首次发现库存 | 尝试快速下单路径 |
| `network_error` | 网络异常 | 尝试重试策略、备用端点 |
| `unexpected_response` | 未知响应格式 | 分析响应结构、发现新字段 |

## 核心组件

### 1. LLM客户端 (ai_lab/llm_client.py)

- 兼容Claude API格式
- 支持自定义API URL
- 错误处理和重试机制
- 超时控制

### 2. 实验执行器 (ai_lab/experiment.py)

- 异步执行，不阻塞主进程
- 收集上下文信息
- 调用LLM生成实验方案
- 执行HTTP实验请求
- 记录结果和改进建议

### 3. 提示词模板 (ai_lab/prompts.py)

- 分析异常的提示词
- 生成实验方案的提示词
- 总结改进建议的提示词

## 安全考虑

1. 实验请求仅限查询类操作，不下单/支付
2. 实验请求有频率限制（每次事件最多3个实验）
3. 实验请求有超时控制（默认5秒）
4. 可通过配置完全禁用AI实验模块

## 实现优先级

1. P0: LLM客户端、实验执行器基础框架
2. P1: 集成到抢购主循环
3. P2: Web界面显示实验结果
4. P3: 自动生成代码改进建议

## 验收标准

1. 抢购遇到限流时，AI能生成并执行实验请求
2. 实验结果记录到日志文件
3. Web界面能查看AI实验分析
4. 主抢购流程不受实验模块影响
