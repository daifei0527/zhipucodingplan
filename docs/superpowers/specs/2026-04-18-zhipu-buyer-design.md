# 智谱 CodingPlan 会员自动抢购脚本设计

## 背景

用户需要在每天10:00自动抢购智谱 CodingPlan 的 Pro 包月会员。手动刷新会被限流，等恢复时名额已售罄。需要一个自动化脚本来提高抢购成功率。

## 目标

- 自动化抢购流程，提高成功率
- 支持无人值守定时运行
- 记录每次运行情况，逐步学习完整抢购流程
- 提供Web界面用于监控和手动处理验证码

## 核心模块

### 1. 登录模块 (auth/)

**功能：**
- 检查Cookie是否有效，有效则跳过登录
- 无效时启动Playwright无头浏览器访问登录页
- 自动填写账号密码
- 出现验证码时，通过Web界面通知用户手动完成
- 登录成功后保存Cookie到本地文件

**输入：** 账号、密码（存储在config.json）
**输出：** 有效的Cookie文件

### 2. 抢购模块 (buyer/)

**功能：**
- 在指定时间（10:00前后）高频发送HTTP请求
- 复用登录模块保存的Cookie
- 检测页面状态：是否被限流、是否有名额
- 发现名额时自动完成下单
- 使用储值余额自动支付

**策略：**
- 提前30秒开始预热（保持Cookie活跃）
- 10:00整开始高频请求
- 遇到限流时按指数退避重试
- 记录每次请求的响应内容供学习模块分析

### 3. 学习模块 (learner/)

**功能：**
- 记录每次请求的URL、参数、响应状态、响应内容
- 保存关键页面的HTML用于分析页面结构
- 自动识别抢购相关的API接口
- 逐步构建完整的抢购流程知识

**输出：** 结构化的日志文件和发现记录

### 4. Web监控界面 (web/)

**功能：**
- 显示当前运行状态（等待中/登录中/抢购中/已完成）
- 出现验证码时嵌入浏览器视图供用户操作
- 显示历史抢购日志
- 提供手动触发抢购的按钮（用于测试）

**端口：** 默认5000，可通过配置修改
**访问方式：** SSH隧道转发到本地访问

## 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 主语言 | Python 3.10+ | 异步支持好，库丰富 |

## 运行流程

1. **启动时**：检查Cookie有效性，无效则触发登录流程
2. **等待抢购时间**：Web界面显示倒计时，后台保持Cookie活跃
3. **抢购时间到**：启动高频请求，记录所有响应
4. **抢购成功**：发送通知，记录结果
5. **抢购失败**：记录原因，等待下次运行

## 安全考虑

- 配置文件包含敏感信息，设置文件权限为600
- Cookie文件同样设置权限为600
- Web界面仅监听localhost，通过SSH隧道访问
