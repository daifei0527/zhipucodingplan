# 多账号抢购功能设计文档

## 概述

为智谱 CodingPlan 自动抢购系统增加多账号管理功能，支持多账号并发抢购、余额判断自动支付、套餐优先级设置，以及 Web 界面管理。

## 需求总结

| 功能 | 描述 |
|------|------|
| 多账号抢购 | 多个账号同时并发抢购，提高抢购成功率 |
| 独立配置 | 每个账号可独立设置目标套餐和优先级 |
| 自动支付 | 余额足够时自动支付，不足时提示用户手动支付 |
| 本地存储 | 使用 JSON 文件存储账号信息 |
| Web 管理 | 支持账号管理、状态监控、手动控制、历史记录 |

## 数据模型

### accounts.json 结构

```json
{
  "accounts": [
    {
      "id": "acc_001",
      "username": "user1@example.com",
      "password": "encrypted_password",
      "enabled": true,
      "target_plans": [
        {"plan": "max", "duration": "monthly", "priority": 1},
        {"plan": "pro", "duration": "monthly", "priority": 2}
      ],
      "auto_pay": true,
      "balance_threshold": 0,
      "created_at": "2026-04-19T10:00:00",
      "last_run": "2026-04-19T10:05:00"
    }
  ],
  "global_config": {
    "max_concurrent": 5,
    "retry_interval": 0.1,
    "purchase_timeout": 300
  }
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 账号唯一标识，格式 acc_XXX |
| `username` | string | 登录用户名 |
| `password` | string | 登录密码（建议加密存储） |
| `enabled` | bool | 是否启用该账号 |
| `target_plans` | array | 目标套餐列表，按 priority 排序 |
| `auto_pay` | bool | 是否自动支付 |
| `balance_threshold` | number | 余额不足阈值，低于此值提示用户 |

## 支付流程

```
抢购成功创建订单
       │
       ▼
 查询账户余额
 (batch-preview API)
       │
       ├─────────────────┐
       ▼                 ▼
  余额 >= 价格       余额 < 价格
       │                 │
       ▼                 ▼
 调用支付API        标记需手动支付
 method=BALANCE     通知用户
       │
       ▼
    支付成功
    记录日志
```

**关键逻辑**：
1. 余额信息从 `batch-preview` API 响应中获取
2. 如果余额不足但订单创建成功，在日志中标记"需手动支付"
3. Web 界面显示需要手动支付的订单

## API 设计

### 账号管理

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/accounts` | GET | 获取账号列表 |
| `/api/accounts` | POST | 添加新账号 |
| `/api/accounts/<id>` | PUT | 更新账号配置 |
| `/api/accounts/<id>` | DELETE | 删除账号 |
| `/api/accounts/<id>/balance` | GET | 查询账号余额 |

### 抢购控制

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/accounts/<id>/start` | POST | 启动单账号抢购 |
| `/api/accounts/<id>/stop` | POST | 停止单账号抢购 |
| `/api/purchase/start` | POST | 启动全部账号抢购 |
| `/api/purchase/stop` | POST | 停止全部抢购 |

### 历史记录

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/history` | GET | 获取抢购历史 |
| `/api/history/<account_id>` | GET | 获取单账号历史 |

### 功能模块

## 并发控制

### 实现方式

- 使用 `asyncio.gather()` 实现多账号并发
- 设置最大并发数限制（默认5个），避免触发服务端限流
- 每个账号独立维护抢购状态和日志

### 错误处理

- 单个账号失败不影响其他账号
- 网络超时自动重试（最多3次）
- 登录失效自动重新登录
- 详细记录失败原因
